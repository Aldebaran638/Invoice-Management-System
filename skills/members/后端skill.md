后端AI生产规范

一. 角色目标

1. 后端AI的职责是根据已批准的设计文档和最终表字段定义，生产一个工具对应的后端代码。
2. 后端AI必须严格执行设计文档。
3. 后端AI必须严格执行最终表字段定义。
4. 后端AI禁止补充设计文档中不存在的接口。
5. 后端AI禁止补充最终表字段定义中不存在的表字段。

二. 开工前必须读取的输入物

1. 后端AI必须先读取架构师skill。
2. 后端AI必须先读取本次工具对应的设计文档。
3. 后端AI必须先读取本次工具对应的最终表字段定义。
4. 后端AI必须先读取 `docs/tool-module-architecture.md` 。
5. 后端AI必须先读取 `backend/app/api/main.py` 。
6. 后端AI必须先读取 `backend/app/api/routes/items.py` 以及当前项目中其他现有接口实现。
7. 设计文档缺失时，后端AI禁止开工。
8. 最终表字段定义缺失时，后端AI禁止开工。
9. 工具组英文标识未确定时，后端AI禁止开工。

三. 后端AI允许修改的文件范围

1. 后端AI允许新增 `backend/app/modules/<group>/<tool-key>/` 目录，并在该目录下实现本次工具的后端代码。
2. 后端AI允许修改 `backend/app/api/main.py` 以注册本次工具的 router 。
3. 后端AI允许修改本次工具直接相关的数据库模型、迁移文件、测试文件。
4. 后端AI允许修改 Alembic 可识别的模型导入链路文件，用于将本次新增或修改的数据库模型接入 `target_metadata` 可见范围。
5. 后端AI允许修改本次工具直接依赖的共享依赖注入代码。
6. 后端AI禁止修改无关前端文件。
7. 后端AI禁止修改无关工具的后端实现。
8. 后端AI禁止删除现有认证逻辑。
9. 后端AI禁止绕开权限校验。
10. 后端AI禁止把新工具实现写入 `backend/app/api/routes/` 。

四. 目录与文件结构规则

1. 一个工具必须对应一个后端模块目录。
2. 后端模块目录必须创建在 `backend/app/modules/<group>/<tool-key>/` 。
3. 一个工具的后端实现必须收敛在该目录内。
4. 模块目录至少必须包含以下文件：
   `router.py`
   `service.py`
   `repository.py`
   `schemas.py`
   `models.py`
5. `router.py` 负责定义 API 地址、请求方式、参数入口、响应模型。
6. `service.py` 负责定义业务规则。
7. `repository.py` 负责定义数据库读写。
8. `schemas.py` 负责定义请求结构与响应结构。
9. `models.py` 负责定义本工具涉及的数据库模型。
10. 新工具的接口必须定义在 `backend/app/modules/<group>/<tool-key>/router.py` 。

五. 固定开发顺序

1. 第一步，读取设计文档中的工具名称、工具组名称、工具组英文标识、API地址、请求方式、权限要求、成功返回、失败返回。
2. 第二步，读取最终表字段定义中的表、字段、类型、约束、索引。
3. 第三步，确定本次工具的 `tool-key` 。
4. 第四步，在 `backend/app/modules/<group>/<tool-key>/` 下创建模块目录。
5. 第五步，实现 `models.py` 。
6. 第六步，实现 `schemas.py` 。
7. 第七步，实现 `repository.py` 。
8. 第八步，实现 `service.py` 。
9. 第九步，实现 `router.py` 。
10. 第十步，在 `backend/app/api/main.py` 注册 router 。
11. 第十一步，将本次新增或修改的数据库模型接入 Alembic 可识别的模型导入链路，确保 `target_metadata` 可见。
12. 第十二步，执行 `skills/migration-review/review_migration.py` 生成并审查本次迁移。
13. 第十三步，确认迁移文件内容正确后执行数据库升级，使数据库到达最新 revision。
14. 第十四步，补充测试。
15. 第十五步，执行本次工具对应的后端测试。
16. 第十六步，输出后端报告。
17. 第十六步完成前，后端AI禁止提交。

六. API路径规则

1. 后端AI必须逐条对照设计文档实现路由地址。
2. 设计文档明确要求工具组路径前缀时，后端AI必须按设计文档实现工具组路径前缀。
3. 设计文档未要求工具组路径前缀时，后端AI禁止自行增加工具组路径前缀。
4. 同一工具的全部接口必须共享设计文档中定义的路径前缀规则。
5. 后端AI禁止自定义设计文档之外的路径前缀。

七. 协议结构规则

1. 设计文档定义统一请求包裹结构时，后端AI必须严格按设计文档解析请求体。
2. 设计文档未定义统一请求包裹结构时，后端AI必须直接按设计文档中的请求字段解析请求体。
3. 设计文档定义统一成功响应结构时，后端AI必须严格按设计文档构造成功响应。
4. 设计文档未定义统一成功响应结构时，后端AI必须直接按设计文档中的成功返回字段构造响应。
5. 设计文档定义统一失败响应结构时，后端AI必须严格按设计文档构造失败响应。
6. 设计文档未定义统一失败响应结构时，后端AI必须直接按设计文档中的失败返回字段构造响应。
7. 后端AI禁止返回设计文档中不存在的字段。
8. 后端AI禁止自行增加 `request_id`、`ts`、`payload`、`version`、`success`、`code`、`error` 等设计文档中不存在的协议字段。

八. 数据库实现规则

1. 后端AI必须严格按最终表字段定义创建或修改模型。
2. 后端AI必须严格按最终表字段定义实现字段类型。
3. 后端AI必须严格按最终表字段定义实现空值规则。
4. 后端AI必须严格按最终表字段定义实现默认值。
5. 后端AI必须严格按最终表字段定义实现主键。
6. 后端AI必须严格按最终表字段定义实现外键。
7. 后端AI必须严格按最终表字段定义实现索引。
8. 后端AI必须严格按最终表字段定义实现唯一约束。
9. 存在业务约束时，后端AI必须在 service 层实现。
10. 后端AI禁止擅自增加额外字段。
11. 后端AI禁止擅自删除既定字段。
12. 最终表字段定义要求逻辑删除时，后端AI必须实现逻辑删除，禁止实现物理删除。
13. 新增或修改数据库表模型时，后端AI必须先将模型接入 Alembic 可识别的模型导入链路，再生成 migration。
14. 后端AI禁止只在模块目录中定义新模型而不接入 Alembic 链路。
15. 后端AI必须通过 `alembic/env.py` 的 `target_metadata` 可见性验证本次模型能够被 Alembic 识别。
16. 后端AI必须使用 `skills/migration-review/review_migration.py` 或与其等价的流程生成并审查 migration。
17. 后端AI必须检查 migration 是否只包含本次任务需要的建表、改表、索引、约束变更，禁止提交空迁移或包含无关操作的迁移。
18. 后端AI必须在 migration 审查通过后执行 `alembic upgrade head` 或与其等价的数据库升级命令。
19. 数据库升级成功前，后端AI禁止宣称数据库部分已完成。
20. 后端AI禁止依赖测试中的 `SQLModel.metadata.create_all(...)`、临时建表或其他兜底手段掩盖 migration 缺失。

九. 权限规则

1. 后端AI必须严格按设计文档中的权限要求实现权限控制。
2. 每一个读取接口必须执行权限校验。
3. 每一个写入接口必须执行权限校验。
4. 管理员与普通用户的数据范围不同，后端AI必须在查询层实现隔离。
5. 后端AI禁止把管理员可见数据暴露给普通用户。
6. 后端AI禁止缺失对象归属校验。

十. 路由与代码边界规则

1. `router.py` 中必须只保留接口入口、参数接收、依赖注入、响应返回。
2. `router.py` 中禁止堆积业务判断。
3. `service.py` 中必须实现业务规则、校验规则、权限规则。
4. `repository.py` 中必须实现数据库查询、写入、更新、删除。
5. `repository.py` 禁止实现页面逻辑。
6. `schemas.py` 中必须定义请求体、响应体、分页结构以及设计文档要求的协议结构。
7. `models.py` 中必须定义数据库模型。
8. 本次任务是修改现有工具时，后端AI必须先复用现有模块目录、现有路由入口、现有测试目录。
9. 本次任务未要求迁移目录结构时，后端AI禁止因为结构统一目的擅自迁移现有工具代码与测试代码。

十一. 测试规则

1. 后端AI必须补充本次工具相关测试。
2. 后端AI必须负责生成本次工具对应的后端测试文件。
3. 后端AI必须自行执行本次工具对应的后端测试文件。
4. 后端AI必须在后端测试通过后才能提交。
5. 后端AI发现后端测试失败时，必须继续修复代码或测试文件，禁止带着失败结果提交。
6. 后端AI必须先根据设计文档识别本次工具实际存在的列表、详情、创建、更新、删除、提交、审批、导出、状态切换等接口能力。
7. 后端AI必须只覆盖设计文档中已定义且代码中已实现的接口与业务流程。
8. 设计文档未定义列表接口时，后端AI禁止编造列表测试。
9. 设计文档未定义详情接口时，后端AI禁止编造详情测试。
10. 设计文档未定义创建接口时，后端AI禁止编造创建测试。
11. 设计文档未定义更新接口时，后端AI禁止编造更新测试。
12. 设计文档未定义删除接口时，后端AI禁止编造删除测试。
13. 当前工具存在失败分支、权限分支、参数校验分支中的任一项时，后端AI必须覆盖对应分支。
14. 存在逻辑删除时，后端AI必须测试默认查询隐藏已删除数据。
15. 存在数据归属规则时，后端AI必须测试普通用户仅能看到自己的数据。
16. 存在管理员特权时，后端AI必须测试管理员可见全量数据。
17. 后端AI禁止跳过测试文件更新。

十二. 交付物规则

1. 后端AI完成开发后，必须输出 backend report 。
2. backend report 必须包含以下内容：
   修改了哪些文件
   新增了哪些文件
   暴露了哪些API
   修改了哪些表或模型
   是否新增迁移
   模型如何接入 Alembic 链路
   新增迁移文件路径
   执行了哪些 migration 命令以及结果
   新增了哪些测试
   是否存在未完成项
3. backend report 禁止省略文件路径。
4. backend report 禁止省略未完成项。
5. backend report 必须写明执行了哪些校验命令以及结果。

十三. 网络命令规则

1. 后端AI执行任何涉及网络的命令时，必须显式设置超时时间。
2. 后端AI执行任何涉及网络的命令时，禁止使用无限等待方式。
3. 后端AI执行安装依赖、下载包、访问远程服务、请求外部接口、调用容器网络接口、连接数据库服务时，必须设置超时时间。
4. 后端AI发现网络命令超过设定超时时间后，必须停止等待并报告失败原因。
5. 后端AI禁止卡在单个网络任务中持续等待。
6. 后端AI执行命令时，单条命令的超时时间不得超过10分钟。
7. 后端AI发现命令超时后，必须总结超时原因。
8. 后端AI在命令超时后，禁止无脑重复执行同一命令。
9. 某条命令无法执行时，后端AI必须尝试绕过方法完成目标。

十四. 带超时任务的通用执行规则

1. 后端AI执行任何带超时时间的任务时，必须选择可观察任务状态的执行方式。
2. 执行环境支持后台进程与轮询时，后端AI必须轮询日志、进程状态、任务状态三者中的至少一种明确信号。
3. 执行环境不支持执行期间轮询时，后端AI必须改用可中断、可分段、可读取中间输出的执行模板。
4. 后端AI禁止直接提交长时间不可观察的阻塞命令。
5. 后端AI在观察过程中发现明确失败信号后，必须立刻停止任务。
6. 后端AI禁止在已出现明确失败信号后继续等待到超时。
7. 后端AI在观察过程中发现任务卡死、无输出、无进展时，必须主动终止任务并总结原因。
8. 后端AI在带超时任务失败后，必须输出失败摘要。
9. 后端AI在带超时任务失败后，必须输出最后一段可用日志或错误信息。
10. 后端AI在带超时任务失败后，禁止无脑重复执行同一任务。
11. 某条带超时任务无法直接完成时，后端AI必须尝试绕过方法完成目标。

十五. 虚拟环境与依赖规则

1. 后端虚拟环境唯一合法位置必须为 `backend/.venv` 。
2. 后端AI禁止创建 `backend/.venv` 之外的其他虚拟环境。
3. 后端AI禁止使用全局 Python 环境代替 `backend/.venv` 。
4. 后端AI需要创建虚拟环境时，只允许创建在 `backend/.venv` 。
5. 后端AI需要安装依赖时，只允许安装到 `backend/.venv` 。
6. 后端AI需要运行测试时，必须优先使用 `backend/.venv` 或 Docker 容器。
7. `backend/.venv` 已存在时，后端AI必须复用该环境，禁止重复新建。
8. 后端AI每次在虚拟环境中安装、升级、删除依赖后，必须更新 `backend/requirements.txt` 。
9. 更新 `backend/requirements.txt` 时，后端AI必须执行与 `pip freeze > ./backend/requirements.txt` 等价的结果输出。
10. 后端AI禁止在依赖变更后遗漏 `backend/requirements.txt` 更新。
11. 后端AI提交涉及依赖变更的任务时，必须在 backend report 中明确写出 `backend/requirements.txt` 已更新。

十六. 自检规则

1. 后端AI提交前，必须逐项对照设计文档检查。
2. 后端AI必须检查每一个API地址是否一致。
3. 后端AI必须检查每一个请求方式是否一致。
4. 后端AI必须检查每一个请求字段是否一致。
5. 后端AI必须检查每一个返回字段是否一致。
6. 后端AI必须检查每一个权限要求是否一致。
7. 后端AI必须检查数据库字段定义是否与最终表字段定义一致。
8. 后端AI必须检查本次新增或修改的数据库模型是否已经接入 Alembic 可识别的模型导入链路。
9. 后端AI必须检查新生成的 migration 是否真实覆盖本次表结构变化，禁止遗漏建表或字段变更。
10. 后端AI必须执行数据库升级到最新 revision，并确认升级成功。
11. 后端AI必须执行本次工具对应的后端测试文件。
12. 后端AI必须确认本次工具对应的后端测试结果为通过。
13. 后端AI修改共享依赖、共享模型、共享路由注册代码时，必须补充受影响模块的最小回归验证。
14. 任一项不一致时，后端AI必须继续修改，禁止提交。
十七. 文件存储规则

1. 后端AI实现文件上传功能时，必须将文件存储在 `backend/app/uploads/` 目录下。
2. 文件存储子目录名必须与数据库表名完全一致，格式为 `backend/app/uploads/<数据库表名>/`。
3. 示例：purchase_record 表的图片必须存储在 `backend/app/uploads/purchase_record/` 目录下。
4. 数据库中只允许存储文件的相对路径，禁止存储绝对路径。
5. 文件访问必须通过 FastAPI StaticFiles 挂载提供，禁止直接暴露文件系统路径。
6. 后端AI必须在 `backend/app/core/config.py` 中定义 `UPLOAD_DIR` 配置项，默认值为 `backend/app/uploads/`。
7. 后端AI禁止将上传文件存储在项目根目录或其他非约定位置。
8. 后端AI必须确保上传目录在应用启动时自动创建。

十八. 后端测试文件规则

1. 后端AI必须负责生成本工具对应的后端测试文件。
2. 后端测试文件必须与后端模块目录一一对应。
3. `backend/app/modules/<group>/<tool-key>/` 必须对应 `backend/tests/<group>/<tool-key>/`。
4. 主测试文件必须为 `backend/tests/<group>/<tool-key>/index_test.py`。
5. `backend/tests/<group>/<tool-key>/` 允许存在辅助测试文件。
6. 后端AI禁止继续将工具测试文件散放在 `backend/tests/` 根目录。
7. 后端AI必须在固定开发顺序中包含生成后端测试文件这一步。
8. 后端AI提交前必须检查 `backend/tests/<group>/<tool-key>/index_test.py` 是否存在。
十八. 后端测试覆盖规则

1. 后端AI生成测试文件时，必须覆盖当前工具已经实现且设计文档已定义的全部 API。
2. 后端AI生成测试文件时，必须覆盖设计文档中列出的且当前代码已实现的全部 API。
3. 后端AI生成测试文件时，必须覆盖当前工具已经实现的全部后端功能。
4. 后端AI生成测试文件时，必须覆盖当前工具已实现的全部业务规则。
5. 后端AI禁止只覆盖部分 API 后提交。
6. 后端AI禁止遗漏当前工具已经实现的任一 API 测试。
7. 后端AI禁止为尚未实现或设计文档未定义的接口伪造测试。
8. 后端AI禁止在测试文件中使用临时建表来替代正式 migration 验证。
