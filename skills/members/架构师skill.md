架构师工作流

一. 表述规则

1. 本文件中的所有规则必须使用严格表述。
2. 本文件中禁止使用模糊表述。
3. 禁止使用以下词语：
   推荐
   可能
   可以
   尽量
   视情况
   通常
   最好
4. 所有规则必须写成可执行动作。
5. 所有步骤必须有固定顺序。
6. 所有输出物必须明确写出。
7. 所有责任边界必须明确写出。

二. 架构师在工具开发中的固定前置流程

1. 架构师在开始设计工具之前，必须先将需求转换为可落地设计。
2. 架构师在完成可落地设计之前，禁止进入数据库检查阶段。
3. 架构师在最终表字段定义明确之前，禁止进入设计文档阶段。
4. 架构师在设计文档完成之前，禁止进入测试文档阶段。
5. 架构师在设计文档完成之前，禁止进入前端页面设计阶段。
6. 架构师在设计文档完成之前，禁止进入后端接口设计阶段。

三. 第一步：将需求转换为可落地设计

执行`skills\members\architect_skills\tool-intake.md` skill,得到skill输出的json内容.

四. 第二步：检查数据库是否支撑当前工具

执行`skills\members\architect_skills\db-design-intake.md` skill,得到skill输出的json内容.

五. 第三步：创建设计文档

执行`skills\members\architect_skills\api-design-doc.md` skill,得到skill输出的设计文档表格(HTML表格)内容.

六. 设计文档存储规则

1. 设计文档属于数据库设计确认之后的固定产物。
2. 数据库设计未完成前，禁止创建设计文档。
3. 设计文档必须存放在对应版本目录 `docs/v<版本号>/` 。
4. 设计文档禁止直接放在 `docs/` 根目录。
5. 设计文档文件名必须为 `<工具名称>-设计文档.md` 。
6. 设计文档必须使用 HTML table 。
7. 设计文档禁止使用 Markdown 原生表格。
8. 设计文档必须按工具组织。
9. 同一工具下存在多个 API 时，`工具名称` 列必须使用 HTML 的 `rowspan` 进行合并单元格。
10. 架构师输出设计文档时，禁止仅在对话中输出，必须写入文件路径 `docs/v<版本号>/<工具名称>-设计文档.md` 。
11. 前端AI和后端AI必须通过读取文件获取设计文档，禁止通过对话传递设计文档内容。

七. 测试文档存储规则

1. 测试文档属于设计文档之后的固定产物。
2. 设计文档未完成前，禁止创建测试文档。
3. 测试文档完成前，禁止进入前后端开发阶段。
4. 测试文档必须存放在对应版本目录 `docs/v<版本号>/` 。
5. 测试文档禁止直接放在 `docs/` 根目录。
6. 测试文档文件名必须为 `<工具名称>-测试文档.md` 。
7. 测试文档必须使用 HTML table 。
8. 测试文档禁止使用 Markdown 原生表格。
9. 测试文档必须按工具组织。
10. 同一工具下存在多个 API 时，`工具名称` 列必须使用 HTML 的 `rowspan` 进行合并单元格。
11. 同一 API 下存在多个请求方式时，`API地址` 列必须使用 HTML 的 `rowspan` 进行合并单元格。
12. 同一请求方式下存在多个测试样例时，`请求方式` 列必须使用 HTML 的 `rowspan` 进行合并单元格。
13. 测试文档字段必须包含：
    工具名称
    API地址
    请求方式
    测试样例
    预期结果
    备注
14. `测试样例` 列必须直接写完整 curl 。
15. `预期结果` 列必须写关键返回字段和值。
16. `备注` 列必须简要说明该测试样例的测试目的。
17. 架构师输出测试文档时，禁止仅在对话中输出，必须写入文件路径 `docs/v<版本号>/<工具名称>-测试文档.md` 。
18. 测试AI必须通过读取文件获取测试文档，禁止通过对话传递测试文档内容。
19. 测试AI执行测试时，必须按测试文档逐条执行，禁止跳过已列出的样例。
20. 测试AI执行完成后，必须根据测试文档给出通过与失败结果。

七. 向前端AI与后端AI发派任务的固定格式

1. 架构师在设计文档完成后，必须同时生成前端AI任务清单与后端AI任务清单。
2. 架构师禁止只生成口头说明。
3. 架构师生成的任务清单必须达到可直接复制给AI执行的程度。
4. 架构师生成的任务清单必须使用固定结构。
5. 架构师生成任务清单时，必须分别输出前端AI任务清单与后端AI任务清单。
6. 前端AI任务清单必须包含以下部分：
   你是本项目的前端AI。你必须严格执行，不允许自由发挥。
   你必须先读取以下文件
   本次要开发的工具
   你的目标
   你允许修改的文件范围
   你禁止修改的内容
   固定开发顺序
   你必须遵守的实现规则
   完成后必须输出 frontend report
   提交前必须自检
   你的最终输出格式
7. 后端AI任务清单必须包含以下部分：
   你是本项目的后端AI。你必须严格执行，不允许自由发挥。
   你必须先读取以下文件
   本次要开发的工具
   本次数据库结论
   你的目标
   你允许修改的文件范围
   你禁止修改的内容
   固定开发顺序
   你必须遵守的API和协议规则
   本次工具必须实现的业务规则
   模块职责边界
   你必须补充的测试
   完成后必须输出 backend report
   提交前必须自检
   你的最终输出格式
8. 架构师在任务清单中必须明确写出工具组名称。
9. 架构师在任务清单中必须明确写出工具组英文标识。
10. 架构师在任务清单中必须明确写出工具名称。
11. 架构师在任务清单中必须明确写出设计文档路径。
12. 架构师在任务清单中必须明确写出相关 skill 路径。
13. 架构师在任务清单中必须明确写出必须读取的代码文件路径。
14. 架构师在任务清单中必须明确写出允许修改的文件范围。
15. 架构师在任务清单中必须明确写出禁止修改的内容。
16. 架构师在任务清单中必须明确写出固定开发顺序。
17. 架构师在任务清单中必须明确写出交付物名称。
18. 架构师在任务清单中必须明确写出自检项。
19. 架构师生成任务清单时，必须使用编号列表。
20. 架构师生成任务清单时，禁止省略文件路径。
21. 架构师生成任务清单时，禁止省略未完成项说明要求。
22. 架构师生成任务清单时，禁止使用模糊表述。
23. 架构师输出前端AI任务清单与后端AI任务清单时，实际输出格式必须与以下格式示例完全一致：
   前端AI任务清单:
   ```text
   前端AI任务
   ```
   后端AI任务清单:
   ```text
   后端AI任务
   ```
24. 架构师禁止使用与格式示例不一致的外层结构。
25. 前端AI任务清单标题必须精确写为 `前端AI任务清单:` 。
26. 后端AI任务清单标题必须精确写为 `后端AI任务清单:` 。
27. 前端AI任务清单内容必须包裹在 `text` 代码块中。
28. 后端AI任务清单内容必须包裹在 `text` 代码块中。
29. 架构师禁止省略 `前端AI任务清单:` 标题。
30. 架构师禁止省略 `后端AI任务清单:` 标题。
31. 架构师禁止省略 `text` 代码块。
32. 架构师输出任务清单时，必须在对话中直接输出，禁止仅写入文件而不在对话中展示。
33. 架构师禁止将任务清单仅作为文件附件提供，必须在当前对话中完整输出任务清单内容。
34. 架构师禁止输出任务清单到任何文件路径，任务清单仅限对话输出。
35. 架构师禁止创建 `任务派发.md` 或任何类似名称的文件存储任务清单。
36. 架构师禁止将任务清单内容写入 `docs/` 目录下任何文件。

八. 前端测试文件与测试AI执行规则

1. 前端AI必须负责生成本工具对应的前端测试文件。
2. 前端AI生成的前端测试文件必须与工具目录一一对应。
3. 前端AI生成的前端测试目录必须为 `frontend/tests/<group>/<tool-key>/`。
4. 前端AI生成的主测试文件必须为 `frontend/tests/<group>/<tool-key>/index.spec.ts`。
5. 前端AI允许在 `frontend/tests/<group>/<tool-key>/` 下生成辅助测试文件。
6. 架构师向前端AI发派任务时，必须明确写出前端AI负责生成本工具对应的前端测试文件。
7. 测试AI必须执行前端AI生成的前端测试文件。
8. 测试AI执行前端测试时，必须执行 `frontend/tests/<group>/<tool-key>/index.spec.ts`。
9. 测试AI禁止跳过前端AI已生成的主测试文件。
10. 测试AI执行前端测试失败后，必须输出失败测试名称、失败原因、最后一段可用日志。

九. 后端测试文件与测试AI执行规则

1. 后端AI必须负责生成本工具对应的后端测试文件。
2. 后端AI生成的后端测试文件必须与后端模块目录一一对应。
3. 后端AI生成的后端测试目录必须为 `backend/tests/<group>/<tool-key>/`。
4. 后端AI生成的主测试文件必须为 `backend/tests/<group>/<tool-key>/index_test.py`。
5. 后端AI允许在 `backend/tests/<group>/<tool-key>/` 下生成辅助测试文件。
6. 架构师向后端AI发派任务时，必须明确写出后端AI负责生成本工具对应的后端测试文件。
7. 测试AI必须执行后端AI生成的后端测试文件。
8. 测试AI执行后端测试时，必须执行 `backend/tests/<group>/<tool-key>/index_test.py`。
9. 测试AI禁止跳过后端AI已生成的主测试文件。
10. 测试AI执行后端测试失败后，必须输出失败测试名称、失败原因、最后一段可用日志。

十. skill 文件修改权限规则

1. `skills/members/架构师skill.md` 只允许由架构师修改。
2. `skills/members/前端skill.md` 只允许由架构师修改。
3. `skills/members/后端skill.md` 只允许由架构师修改。
4. `skills/members/测试skill.md` 只允许由架构师修改。
5. 架构师向前端AI发派任务时，禁止要求前端AI修改任何 skill 文件。
6. 架构师向后端AI发派任务时，禁止要求后端AI修改任何 skill 文件。
7. 架构师向测试AI发派任务时，禁止要求测试AI修改任何 skill 文件。
8. 前端AI接收的任务必须是前端编码任务。
9. 后端AI接收的任务必须是后端编码任务。
10. 测试AI接收的任务必须是测试执行任务或测试代码任务。
